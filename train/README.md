# 고스트 어댑터 (zero-init residual adapter) PoC

Whisper를 **건드리지 않은 채** 위에 얇은 보정 레이어를 얹어, 틀리는 케이스만
고치고 맞는 케이스·다른 언어는 보존하는 실험. 전체 재학습(불가능)도, 위험한
전체 파인튜닝(영어 붕괴)도 아닌 중간 길.

## 원리

- 디코더 출력과 lm_head 사이에 N겹 잔차 블록을 끼움: `x = x + Wup(GELU(Wdown(LN(x))))`
- **Wup을 0으로 초기화** → 시작 시 delta=0 → **원본과 비트 단위 동일 출력**
  (ControlNet zero-conv, LoRA B-0init과 같은 원리. `verify_identity.py`로 증명됨)
- base 전부 동결, 고스트 블록만 학습 (large-v3-turbo 기준 전체의 ~2%)
- **리허설**: 맞은 케이스도 정답과 함께 학습 → catastrophic forgetting 방지

이 두 가지(0-init + 리허설)가 ghost613 파인튜닝이 실패한 원인(영어 붕괴·과적합)을
구조적으로 막는다.

## 검증된 것 (이 PC, CPU)

- `verify_identity.py`: 0-init 고스트 = 원본과 출력 차이 **0.00e+00** (항등 보장 성립),
  가중치 교란 시 출력 변함 (학습 가능). **PASS**
- `poc_adapter.py` 스모크(whisper-tiny): 파이프라인 정상 동작, loss 하강 확인.
  낮은 LR에서 보존 목표(깨끗한 음성·영어 유지) 성립.
- **단**, 실제 "잡음 보정" 효과는 whisper-tiny로는 검증 불가 (한국어를 거의 못하는
  모델 + CPU 학습 한계). 아래 GPU 실전 설정 필요.

## 실전 결과 (RTX 4090, large-v3-turbo, FLEURS held-out)

세 번의 실험으로 설계를 입증했다. 모든 수치는 학습에 쓰지 않은 held-out 발화 기준:

| 실험 | 잡음 한국어 CER | 깨끗 한국어 CER | 영어 WER | best | 비고 |
|---|---|---|---|---|---|
| v1 (어댑터를 lm_head 직전) | 180%로 붕괴 | 180%로 붕괴 | 보존 | step 0 | 출력 직전 배치가 자기회귀 생성에 치명적 |
| v2 (디코더 레이어 내부) | 개선 신호 | 개선 신호 | +1p 드리프트 | step 0 | 안정화됐으나 영어 퇴보로 벌점 거부 |
| **v3 (+영어 리허설)** | **12.6 → 11.0** | **9.9 → 8.6** | **5.2 → 5.0** | **step 1000** | **3지표 동시 달성** |

**결론**: 0-init 고스트 어댑터를 **디코더 레이어 내부**에 두고, 한국어 보정 +
**한국어·영어 리허설**을 함께 학습하면, held-out에서 잡음 한국어를 상대 13%
개선하면서 깨끗한 한국어·영어를 보존(오히려 미세 개선)할 수 있다. early-stopping이
과적합 시점(step 1000 이후) 전의 best를 정확히 선택한다.

두 가지가 결정적이었다: (1) 어댑터를 출력 직전이 아닌 레이어 내부에 배치(붕괴 방지),
(2) 영어 리허설로 catastrophic forgetting 차단. 320 → 400 예제(한국어 160 리허설 +
160 잡음보정 + 영어 80 리허설)의 소규모 PoC로, 데이터를 늘리면 개선폭이 커질 여지가 있다.

## v4 — 다국어 배터리로 드러난 진실 (중요한 반전)

v3의 "성공"은 **평가가 좁았던 착시**였다. FLEURS 영어 하나만 봤기에 멀쩡해 보였지만,
원본 분포 전체(다국어)를 측정하니 퇴보가 드러났다. `poc_distill.py` +
다국어 배터리(ko/en/ja/zh/de/fr) 결과 (RTX 4090):

| 분포 | base | 학습 중 관측 |
|---|---|---|
| 잡음 ko | 14.2 | 개선 안 됨 (14~26 진동) |
| en, fr | 7.8 / 7.0 | 유지 |
| **ja** | 8.1 | **10~25로 심하게 퇴보** |
| **zh** | 10.3 | **17~34로 심하게 퇴보** |
| de | 10.7 | 11~14 약간 퇴보 |

`best = step 0` — 한국어를 개선하면서 다른 언어를 안 망치는 체크포인트가 **없어서**
시스템이 원본을 그대로 반환했다.

**원인**: 한·일·중은 문자/토큰을 공유 → 한국어 보정 압력이 공유 토큰을 끌어당겨
일·중을 망가뜨린다. 영어는 토큰이 멀어 영향을 안 받아 v3에서 안 보였던 것.
다국어 시퀀스 증류 앵커(언어당 40개)로도 이 간섭을 못 막았다.

**교훈**: "우리 테스트셋에서 좋아져도 원본 분포에선 나빠질 수 있다"가 실측으로
확인됐다. 좁은 평가의 위험을 보여주는 사례. 이후 모든 실험은 6분포 배터리로 검증한다.

## v5 — 언어 게이트로 해결 (연구 목표 달성)

v4의 교훈: 보존을 손실/증류로 "유도"하면 CJK 간섭을 못 막는다. v5는 **아키텍처로
보장**한다 — 고스트를 한국어일 때만 켜고 다른 언어는 우회(`set_ghost_active`).
학습은 한국어 전용, Whisper의 언어 식별로 추론 시 게이트.

`poc_gated.py` 결과 (RTX 4090, 6분포 배터리):

| 분포 | base | +게이트고스트 | 변화 |
|---|---|---|---|
| 잡음 한국어 | 14.2 | **13.3** | **-0.9 (개선)** |
| 깨끗 한국어 | 10.5 | **9.7** | **-0.8 (개선)** |
| en / ja / zh / de / fr | 7.8 / 8.1 / 10.3 / 10.7 / 7.0 | 동일 | **+0.0 (보존 보장)** |

`best = step 200`. **모든 학습 스텝에서** 비한국어 5개 언어가 base와 비트 단위 동일
(우회하므로). v4에서 8→25로 붕괴했던 일본어가 v5에선 정확히 8.1 유지.

**결론 — 연구 질문에 대한 답**: 원본 데이터 없이, 원본 분포를 *보장* 수준으로
유지하면서 새 분포(잡음 한국어)를 개선할 수 있다. 핵심은 보존을 손실로 유도하지
않고 **언어 게이트로 구조적으로 보장**하는 것.

### 한계·배포 주의 (게이트 방식)

- **언어 식별에 의존**: 추론 시 Whisper가 먼저 언어를 감지하고 한국어면 게이트 ON.
  오감지로 *비한국어를 한국어로* 판정하면 그 입력에 어댑터가 적용돼 손상 가능
  (반대 오감지는 안전 — 보정만 누락). 한국어 화자 환경에선 위험 낮음.
- 개선폭은 소규모 PoC 기준 ~1p. 실제 회의 오류·정답 쌍을 대량 확보하면 커질 여지.
- 전체 실험 경로(v1~v5)는 train/poc_*.py 와 git 로그 참조.

### 배포 시 주의

고스트 어댑터는 PyTorch/transformers 모델이다. 우리 추론 스택(faster-whisper=CTranslate2,
OpenVINO)은 표준 Whisper 구조만 받으므로, 어댑터가 들어간 모델을 그대로 변환할 수 없다.
배포하려면 (a) transformers로 추론(느림), 또는 (b) 어댑터를 접어 표준 구조로 재구성하는
추가 작업이 필요하다.

## GPU PC에서 실전 실행

```powershell
# 1) CUDA torch + 학습 의존성 (GPU PC venv)
uv pip install torch transformers --index-url https://download.pytorch.org/whl/cu128
uv pip install soundfile librosa

# 2) 데이터 준비 (없으면)
python test_audio\fetch_datasets.py 200          # FLEURS ko 200개
# (영어는 fetch_fleurs(N,"en_us","fleurs_en")로)

# 3) 항등 성질 먼저 확인
python train\verify_identity.py

# 4) 실전 학습 — large-v3-turbo, 충분한 데이터/스텝
python train\poc_adapter.py `
    --model openai/whisper-large-v3-turbo `
    --train 200 --heldout 40 --steps 2000 --lr 3e-4 --layers 4 `
    --snr 0 --save train\ghost_turbo_ko_noisy.pt
```

## 결과 해석 (held-out 3지표)

| 지표 | 목표 | 의미 |
|---|---|---|
| 잡음 한국어 CER | **↓ 내려감** | 보정 성공 |
| 깨끗 한국어 CER | = 유지 | 맞던 것 안 망가짐 |
| 영어 WER | = 유지 | forgetting 없음 |

**세 개가 동시에 만족돼야 성공.** 잡음 CER만 내려가고 나머지가 오르면 과적합 →
LR을 낮추거나(1e-4), 리허설 비중을 늘리거나, 학습 데이터를 늘릴 것.

## 주의 (실험 설계의 함정)

1. **과적합/암기**: 학습 발화가 적으면 일반화 대신 그 샘플을 외움. 반드시
   held-out(학습에 안 쓴 발화)으로 측정. 스모크에서 tiny+높은 LR이 발산한 게 그 예.
2. **추론 스택 분리**: faster-whisper/OpenVINO는 추론 전용(학습 불가). 학습은
   PyTorch transformers로, 배포는 고스트를 병합 후 CT2/OV로 변환.
3. **데이터가 본질**: 가장 가치 있는 건 *실제 사용 중 Whisper가 틀린 케이스 +
   사람이 고친 정답*. FLEURS는 이미 정확도가 높아 보정할 거리가 적다. 이 기법은
   범용 개선이 아니라 **도메인 특화 보정**(고유명사·전문용어·잡음)에서 빛난다.
